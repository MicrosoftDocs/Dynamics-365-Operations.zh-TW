# <a name="cost-control-workspace"></a>成本控制工作區 

[!include [banner](../includes/banner.md)]

**成本控制** 工作區是一個中心點，負責控制一個維度內或跨維度 (例如，成本中心和產品群組) 的成本物件或一組成本物件的經理可以存取報表。 工作區中的報表完全由成本會計師管理，因此用於報表的配置和資料可以在整個組織中保持一致。

## <a name="cost-control-workspace-configuration"></a>成本控制工作區設定

成本會計師可以根據所需的資料組合或配置定義任意數量的報表設定。 報表設定由六個部分組成，每個部分都有助於選擇目標資料組合或配置。

要設定成本控制工作區，請點選 **成本會計** \> **設定** \> **成本控制工作區設定**。

### <a name="general"></a>一般

在 **一般** FastTab，您可以建立獨特的報表配置。 報表的名稱將是一個使用者將能在 **成本控制** 工作區中進行識別的唯一識別碼。 您還可以指定報表是否應共用或供成本會計師內部使用。

| 欄位       | 描述 |
|-------------|-------------|
| 姓名        | 輸入配置的唯一名稱。 |
| 描述 | 輸入詳細描述。 |
| 已發佈   | 如果將此欄位設定為 **是**，獲得以下角色之一的使用者可以在 **成本控制** 工作區中看到報表：<ul><li>成本會計經理</li><li>成本會計</li><li>成本會計員</li><li>成本物件控管者</li></ul>如果將此欄位設定為 **否**，則僅獲得以下角色之一的使用者可以在 **成本控制** 工作區中看到報表：<ul><li>成本會計經理</li><li>成本會計</li><li>成本會計員</li></ul> |

### <a name="data-filtering"></a>資料篩選

在 **資料篩選** FastTab 中定義報表的資料基礎。 此報表的使用者將在處理來源資料後看到報表中的值。

| 欄位                                                             | 描述 |
|-------------------------------------------------------------------|-------------|
| 成本會計分類帳                                            | 做為報表基礎的 **成本會計分類帳**。 該值來源於 **成本控制單元** 欄位。 |
| 成本控制單位                                                 | 您選擇的值決定了將作為此報表基礎的成本會計分類帳和成本物件。 |
| 統計維度階層、成本元素維度階層 | **成本控制** 工作區設定記錄可以報表非貨幣或貨幣值，但不能在相同的配置中。 在 **成本元素維度階層** 欄位選擇一個值，以報告貨幣值。 在 **統計維度階層** 欄位選擇一個值，以報告非貨幣值。 您選擇的維度階層記錄決定了報表和彙總等級的結構。<blockquote>[!NOTE]<br>要並排查看非貨幣值和貨幣值，您可以為了 Microsoft Power BI 內容套件將資料匯出到 Microsoft Excel。</blockquote> |
| 成本物件維度階層                                   | 選擇適合您定義的報表用途的成本物件維度維度階層。 |
| 預算原始版本                                           | 選擇做為此報表內容中原始預算的預算版本識別碼。 |
| 預算修訂版本                                            | 選擇做為此報表內容中修訂預算的預算版本識別碼。 |

### <a name="assign-calculation-records"></a>指派計算記錄

間接費用計算對來源資料執行多個計算步驟，例如成本行為分類、成本分配和成本分攤。 可以對同一會計期間進行多次間接費用計算，以防發現缺少來源資料或必須更新規則。 每個間接費用計算都使用唯一識別碼儲存。 成本會計可以選擇特定的間接費用計算識別碼。 報表的使用者 (例如經理) 將在 **成本控制** 工作區看到間接費用計算的結果。

| 欄位                  | 描述 |
|------------------------|-------------|
| 會計行事曆期間 | 選擇要為其分配間接費用計算識別碼的會計行事曆期間。<blockquote>[!NOTE]<br>該欄位中列出的會計期間來自與成本會計分類帳關聯的會計行事曆。</blockquote> |
| 實際版本         | 選擇適當的間接費用計算識別碼。 |
| 預算版本         | 選擇適當的間接費用計算識別碼。 |
| 修訂後的預算版本 | 選擇適當的間接費用計算識別碼。 |

### <a name="fiscal-periods-per-column"></a>每欄的會計期間

**每欄的會計期間** FastTab，成本會計師決定應在報表配置中顯示的會計期間。

所選欄中的值將乘以 **每欄的會計期間** FastTab 上的選定值。

| 欄位                | 描述 |
|----------------------|-------------|
| 目前期間       | 顯示目前會計期間的餘額。<blockquote>[!NOTE]<br>根據預設，目前期間由工作階段日期確定。 在 **成本控制** 工作區，可以選擇特定的會計期間。 然後，所選值代表目前期間。</blockquote> |
| 上一個期間      | 顯示上一個會計期間的餘額。 使用下列公式：<br>目前會計期間 - 1<blockquote>[!NOTE]<br>根據預設，上一個期間衍生自工作階段日期。 在 **成本控制** 工作區，可以選擇特定的會計期間做為目前期間。 然後將重新計算相應的 **上一個期間**。</blockquote> |
| 年初至今         | 顯示了年初至今的情況。 使用下列公式：<br>YearToDate (目前會計期間)<blockquote>[!NOTE]<br>根據預設，目前期間由工作階段日期確定。 在 **成本控制** 工作區，可以選擇特定的會計期間。 所選值代表目前期間，而 **年初至今** 值將相應更新。</blockquote> |
| 年初至今平均值 | 顯示了年初至今平均的情況。 使用下列公式：<br>(YearToDate [目前會計期間]) ÷ (計數 [目前會計期間])<p><strong>範例</strong></p><ul><li>**統計維度成員：** 全職員工</li><li>**目前日期：** 3-21-2017</li><li>**期間：** 會計期間 1、會計期間 2、會計期間 3</li><li>**量值：** 10、10、12</li></ul>在這種情況下，**年初至今平均**= (10 + 10 + 12) ÷ 3 = 10.67<p>可以計算成本元素維度成員和統計維度成員的 **年初至今平均** 值。</p><blockquote>[!NOTE]<br>根據預設，目前期間由工作階段日期確定。 在 **成本控制** 工作區，可以選擇特定的會計期間。 所選值代表目前期間，而 **年初至今** 和 **年初至今平均** 值將相應更新。</blockquote> |

### <a name="columns-to-display-for-costs"></a>針對成本顯示的欄

在 **為成本顯示的欄** FastTab，成本會計決定報表配置應包含哪些欄。 分為三類：固定成本、變動成本和未分類成本。

| 欄位                 | 描述 |
|-----------------------|-------------|
| 固定成本            | 此欄類型根據選定的間接費用計算識別碼顯示固定成本。<blockquote>[!NOTE]<br>僅當為會計期間選擇間接費用計算識別碼時，此欄類型才會顯示餘額。</blockquote> |
| 變動成本         | 此欄類型根據選定的間接費用計算識別碼顯示變動成本。<blockquote>[!NOTE]<br>僅當為會計期間選擇間接費用計算識別碼時，此欄類型才會顯示餘額。</blockquote> |
| 固定 + 變動成本 | 此欄類型根據選定的間接費用計算識別碼顯示固定成本和變動成本。<blockquote>[!NOTE]<br>僅當為會計期間選擇間接費用計算識別碼時，此欄類型才會顯示餘額。</blockquote> |
| 總成本            | 此欄類型顯示總成本 (未分類成本、固定成本和變動成本)。<blockquote>[!NOTE]<br>欄類型將始終顯示餘額。</blockquote> |
| 未分類成本     | 此欄類型顯示未分類成本。<blockquote>[!NOTE]<br>此欄可用於驗證是否所有成本都已透過間接費用計算正確分類，或者是否必須調整成本行為規則。</blockquote> |

### <a name="columns-to-display-for-budgeted-costs"></a>針對預算成本顯示的欄

在 **為預算成本顯示的欄** FastTab，成本會計應為選定的預算版本顯示哪些欄。 可以對原始預算和修訂預算進行單獨選擇。

> [!NOTE]
> 由於以下欄位對於原始預算和修訂預算的行為相同，因此將僅解釋一次。

| 欄位                     | 描述 |
|---------------------------|-------------|
| 預算                    | 預算餘額將按所選欄顯示。<blockquote>[!NOTE]<br>餘額將以在 **資料篩選** FastTab 上選擇的預算版本為基礎。</blockquote> |
| 預算差異           | 計算並顯示預算與實際值之間的差異。 使用下列公式：<br>預算餘額 - 實際餘額 |
| 預算差異 %      | 計算並顯示預算與實際值之間的差異百分比。 使用下列公式：<br>(預算餘額 - 實際餘額) ÷預算餘額 |
| 差異期間閾值 | 為目前期間的貨幣金額差異設定閾值。 如果超過閾值，將以紅色醒目顯示 **成本控制** 工作區中的行。<blockquote>[!NOTE]<br>此欄位僅適用於代表支出的成本元素。</blockquote> |
| 差異年閾值   | 為當年的貨幣金額差異設定閾值。 如果超過閾值，將以紅色醒目顯示 **成本控制** 工作區中的行。 |
| 差異閾值 %      | 設定百分比差異的閾值。 如果超過閾值，將以紅色醒目顯示 **成本控制** 工作區中的行。<blockquote>[!NOTE]<br>相同的百分比閾值適用於目前期間和年份。</blockquote> |

## <a name="cost-control-workspace"></a>成本控制工作區

**成本控制** 工作區設計為一個 Web 報表。 因此，所有負責成本物件的經理都可以被授予[定義成本物件控制者的存取權限](access-rights-cost-object-controller.md)中說明的存取權。

可供使用者 (例如經理) 使用的報表清單由設定 **成本控制工作區設定** 頁面上的 **已發佈** 選項進行控制。

![使用者可以在成本控制工作區中看到的報表。](./media/report-cost-control.png)

經理可以選擇要查看的會計行事曆期間。 工作階段日期用於確定預設的目前期間。

會計行事曆期間的值由與 **成本控制工作區設定** 頁面上的報表名稱相關聯的成本會計分類帳的報表名稱和會計行事曆確定。

在成本物件維度階層中，使用者可以選擇應顯示餘額的彙總等級。 透過啟用存取等級安全性，您可以控制權限，以便使用者只能選擇他們已被授予存取權限的階層等級。 因此，他們只能看到已獲得存取權的匯總餘額。

### <a name="add-or-remove-columns"></a>新增或移除欄

使用者可以自訂報表上的欄以滿足他們的要求。

### <a name="view-details"></a>檢視詳細資料

使用者可以深入了解工作區中顯示的餘額背後的詳細資料。 如果使用者選擇成本元素維度階層節點，然後點選 **查看詳細資料**，**成本元素詳細資料** 對話方塊將顯示節點的詳細資料。

格線顯示與成本元素維度階層節點關聯的每個成本元素及其值。 格線中顯示的欄與工作區設定相符。

兩個圖表按期間顯示實際與預算對比和預算差異的摘要。

![圖表按期間顯示實際與預算對比和預算差異的摘要。](./media/cost-element-details-operations.png)

使用者可以點選 **成本分錄** 以根據需要深入了解分錄詳細資料。

![成本分錄。](./media/cost-entries.png)

例如，租金是分配給成本中心的支出。 想要了解其成本中心必須承擔的租金成本的使用者可以深入了解租金是如何計算的。

如果使用者點選 **成本分錄** 頁面的 **分配基礎**，出現一個對話方塊。 然後，使用者可以將分配基礎分配給規則，並查看為該期間註冊的相應統計量值。

在以下範例中，分配基礎是 **公式分配基礎** 類型，並顯示公式。 列出了定義公式的係數。 此外，格線會顯示每個成本物件完成的計算。

![每個成本物件的計算。](./media/cost-entries-allocation-base.png)
