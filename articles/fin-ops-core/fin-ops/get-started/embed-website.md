# <a name="embed-third-party-apps"></a>嵌入第三方應用程式

[!include [banner](../includes/banner.md)]

許多客戶使用一系列應用程式來執行他們的業務。 其中一些應用程式是與財務和營運應用程式結合使用的第三方 Web 應用程式。 若要提供更無縫的使用者體驗，您可以使用 **整頁應用程式** 功能將這些第三方應用程式直接嵌入到您的財務和營運應用程式中 (前提是第三方應用程式允許自己嵌入)。 透過這種方式，使用者可以存取他們需要的網站和應用程式，而無需切換索引標籤或窗口。

在您可以將第三方應用程式嵌入到產品中之前，您必須打開功能管理中的 **整頁應用程式** 功能。 然後，您可以使用以下方法之一嵌入第三方應用程式或網站。 這些方法類似於用於將畫布應用從 Microsoft Power Apps 嵌入到財務和營運應用程式中的方法。

- 將應用程式或網站作為新索引標籤頁 (樞紐索引標籤、FastTab、刀鋒或工作區部分) 嵌入現有頁面。
- 從儀表板為應用程式或網站建立新的全頁體驗。

## <a name="embed-a-website-on-an-existing-page"></a>在現有頁面上嵌入網站

如果您想用嵌入式應用程式補充系統中的現有頁面，請使用此程序。

1. 打開要嵌入應用程式的頁面。
2. 開啟 **新增應用程式** 窗格：

    1. 選取 **設定**，然後 **個人化** 以開啟 **個人化** 工具列。
    2. 選取 **更多\>新增應用程式**。

3. 選取要新增應用程式的頁面區域。 此區域必須是樞紐索引標籤、FastTab、刀鋒或工作區部分的 *索引標籤容器*。
4. 選取 **網站**。
5. 設定嵌入式應用程式：

    - **名稱** – 輸入應為包含嵌入式應用程式的索引標籤顯示的文字。 通常，您會希望此文字成為應用程式的名稱。
    - **URL** – 指定應用程式的位置。

    > [!IMPORTANT]
    > - 出於安全性原因，URL 必須使用安全超文字傳輸安全協定 (HTTPS)。
    > - 應用程式或網站必須設定為允許嵌入。

6. 選取 **儲存** 在頁面上嵌入應用程式。 該應用程式被新增為組中的最後一個索引標籤或部分。
7. 確認應用程式按預期顯示。 如果應用程式未呈現，請參閱本主題後面的[疑難排解](#troubleshooting)章節。
8. 開啟視圖選取器，然後選取 **儲存** (如果應用程式應該與現行視圖相關) 或 **另存為** (將應用程式儲存到不同的視圖)。

    如果頁面沒有視圖選取器 (例如，如果頁面是對話方塊或工作區)，則可以跳過此步驟。

## <a name="embed-a-website-as-a-full-page-experience-from-the-dashboard"></a>從儀表板嵌入網站作為全頁體驗

如果您要嵌入的應用程式與現有頁面無關，或者您只想在財務和營運應用程式中獲得應用程式的全頁體驗，請使用此程序。

1. 開啟儀表板。
2. 在儀表板上選取並按住 (或按右鍵點選)，選取 **個人化**，然後選取 **新增頁面**。
3. 在 **新增頁面** 窗格中，選取 **網站**。
4. 設定嵌入式應用程式：

    - **名稱** – 輸入應顯示在為儀表板上的嵌入式應用程式新增的圖格上的文字。 通常，您會希望此文字成為應用程式的名稱。
    - **URL** – 指定應用程式的位置。

    > [!IMPORTANT]
    > - 出於安全性原因，URL 必須使用 HTTPS。
    > - 應用程式或網站必須設定為允許嵌入。

5. 選取 **儲存** 以將應用程式作為新圖格新增到儀表板。
6. 選取儀表板上的新磁貼，並確認應用程式按預期顯示。 如果應用程式未呈現，請參閱本主題後面的[疑難排解](#troubleshooting)章節。

## <a name="sharing-embedded-apps"></a>分享嵌入式應用程式

使用前面部分中描述的方法之一嵌入應用程式後，您可能希望與系統中的其他使用者分享視圖。 若要分享嵌入式應用程式，請使用以下方法之一：

- **發布視圖 (推薦)**：如果嵌入式應用程式已儲存到視圖中，則推薦和偏好的分享方式是將視圖發布給在目標法律實體中具有適當安全角色的使用者。 在這種情況下，只有所需的使用者才能在該頁面上看到嵌入式應用程式。 如需如何發布視圖的相關資訊，請參閱[發布視圖](saved-views.md#publishing-views)。

    您還可以從儀表板發布已作為全頁體驗嵌入的應用程式。 在儀表板上，選取並按住（或右鍵點選）與應用程式關聯的圖格，選取 **個人化**，然後選取 **發布頁面**。 顯示類似於 *發布視圖* 體驗的體驗，您可以選取要發佈到的安全角色。 在更新 10.0.21 或更高版本中，如果打開了 **改進的對已儲存視圖的法律實體支援** 功能，您還可以將應用程式發佈到所需的法律實體。

- **複製個人化：** 對於不支援視圖的頁面 (例如，對話方塊或工作區)，或者對於全頁應用程式體驗，您可以將個人化設定複製給相應的使用者。 如需詳細資訊，請參閱[分享個人化](personalize-user-experience.md#sharing-personalizations)。

## <a name="viewing-embedded-apps"></a>檢視嵌入式應用程式

若要在財務和營運應用程式的頁面上查看嵌入式應用程式，請打開包含嵌入式應用程式的頁面。 請記住，在某些頁面上，可以使用標準操作窗格上的 **Power Apps** 按鈕存取嵌入式應用程式。 或者，它們可以作為新索引標籤、FastTab、刀鋒或工作區中的新部分直接顯示在頁面上。

## <a name="editing-or-removing-embedded-apps"></a>編輯或刪除嵌入式應用程式

將應用程式嵌入頁面後，您可能必須編輯其設定 (例如，透過更改部分標籤或 URL)。 或者，您可能必須將其從頁面中刪除。 使用以下程序之一編輯嵌入式應用程式的設定或將其完全刪除。

### <a name="apps-that-are-embedded-on-existing-pages"></a>嵌入現有頁面的應用程式

1. 打開嵌入應用程式的頁面。
2. 選取 **設定**，然後 **個人化** 以開啟 **個人化** 工具列。
3. 選取 **選取** 工具，然後選取嵌入式應用程式。
4. 若要編輯應用程式，對其設定進行必要的更改，然後選取 **儲存**。

    或者，若要刪除應用程式，請選取 **刪除**。

5. 重新儲存或重新發布視圖。 請注意，如果您在未明確儲存視圖的情況下離開頁面，則不會保留您在 **編輯網站** 窗格中執行的任何操作。

### <a name="apps-that-are-embedded-from-the-dashboard"></a>從儀表板嵌入的應用程式

1. 開啟儀表板。
2. 選取並按住 (或按右鍵點選) 與嵌入式應用程式相關的圖格，然後選取 **個人化**。
3. 若要編輯應用程式，請選取 **編輯頁面**。 在 **編輯網站** 窗格中，對應用設定進行必要的更改，然後選取 **儲存**。

    或者，若要刪除應用程式，請選取 **移除頁面**。

## <a name="appendix"></a>附錄

### <a name="troubleshooting"></a>排除障礙

如果網站在嵌入到財務和營運應用程式後未正確呈現，或者如果您收到指出 URL 被拒絕連接的錯誤訊息，則該網站可能已設定為阻止其自身嵌入 iframe。 按照以下步驟確定是否可以嵌入網站。

1. 打開您正在使用的瀏覽器的開發人員工具。
2. 在 **網路** 索引標籤上，尋找並選取來自嵌入式站點的回應。
3. 在 **標頭** 索引標籤上的 **回應標頭** 下，尋找 **x-frame-options**。 如果 **x-frame-options** 存在於回應標頭中，其值為 **DENY** 或 **SAMEORIGIN**，則該網站現行無法嵌入。 您必須與應用程式的擁有者合作，才能安全地嵌入它。

### <a name="developer-modeling-a-website-on-a-form"></a>[開發人員] 在表單上建模網站

雖然本主題側重於透過個性化嵌入第三方應用程式或網站，但開發人員也可以使用 Visual Studio 開發體驗將它們嵌入到表單中。 只需在表單中新增一個 **WebsiteHostControl** 控制項。 可用於控制項的中繼資料屬性提供與個人化體驗相同的功能。
