# <a name="number-sequences-overview"></a>編號順序概觀

[!include [banner](../includes/banner.md)]

編號規則用於為需要標識符的主資料記錄和交易記錄產生可讀的唯一識別碼。 需要識別碼的主資料記錄或交易記錄稱為 *參考*。

在為參考建立新記錄之前，您必須設定數列並將其與參考相關聯。 我們建議您使用 **組織管理** 中的頁面來設定數列。 如果需要特定於模組的設定，您可以使用模組中的參數頁面為該模組中的參考指定數列。 例如，在 **應收帳款** 和 **應付帳款**，您可以設定數列組以將特定數列指派給特定客戶或廠商。

設定數列時，您必須指定一個範圍，該範圍定義了哪個組織使用該數列。 該範圍可以是 **共用**、**公司**、**法律實體**，或 **營運單位**。 **法律實體** 和 **公司** 範圍可以與 **會計行事曆期間** 建立更具體的數列。

數列格式由區段組成。 範圍不是 **共用** 的數列可以包含與範圍對應的區段。 例如，範圍為 **法律實體** 的數列可以包含法律實體區段。 透過在數列格式中包含範圍區段，您可以透過查看其編號來識別特定記錄的範圍。

除了與範圍相對應的區段之外，數列格式還可以包含 **常數** 和 **英數字元區段**。 **常數** 區段包含一組不變的字母、數字或符號。 **英數字元** 區段包含一組字母或數字，每次使用數字時都會增加。 使用數字符號 (\#) 表示遞增的數字，和 & 符號表示遞增的字母。 例如，格式 \#\#\#\#\#\_ 2017 建立序列 00001\_2017、00002\_2017 等。

## <a name="number-sequence-examples"></a>數列範例

以下範例展示了如何使用段來建立數列格式。 特別是，這些範例展示了使用範圍區段的效果。

### <a name="expense-report-numbers"></a>費用報表編號

在以下範例中，為名為 **CS** 的法律實體設定了費用報表編號。

- **區域：** 差旅與費用
- **參考：** 費用報表編號
- **範圍：** 法律實體
- **法律實體：** CS

| 區段  | 區段類型 | 值     |
|-----------|--------------|-----------|
| 區段 1 | 法律實體 | CS        |
| 區段 2 | 常數     | -EXPENSE- |
| 區段 3 | 英數字元 | \#\#\#\#  |

**格式化數字範例**：CS-EXPENSE-0039

您可以為其他法律實體設定類似的數列格式。 例如，對於名為 **RW** 的法律實體，如果僅更改法律實體段的值，則格式化數字為 RW-EXPENSE-0039。 您還可以更改其他法律實體的完整數列格式。 例如，您可以省略法律實體範圍段以建立格式化編號，例如 Exp-0001。

### <a name="sales-order-numbers"></a>銷售訂單編號

在以下範例中，為公司 ID **CEU** 設定了銷售訂單編號。

- **區域：** 銷售
- **參考：** 銷售訂單
- **範圍：** 公司
- **公司：** CEU

| 區段  | 區段類型 | 值    |
|-----------|--------------|----------|
| 區段 1 | 常數     | SO-      |
| 區段 2 | 英數字元 | \#\#\#\# |

**格式化數字範例**：SO-0029

即使格式中不包含範圍區段，也會為每個公司 ID 重新編號。 如果您對所有公司 ID 使用相同的格式，則每個公司都使用相同的編號。 例如，每個公司都使用銷售訂單編號 SO-0029。 您還可以更改其他公司 ID 的完整數列格式。

### <a name="purchase-requisition-numbers"></a>採購申請編號

在以下範例中，採購申請編號是組織範圍的。

- **區域：** 購買
- **參考：** 採購申請
- **範圍：** 共用

| 區段  | 區段類型 | 值    |
|-----------|--------------|----------|
| 區段 1 | 常數     | 申請      |
| 區段 2 | 英數字元 | \#\#\#\# |

**格式化數字範例**：Req0052

因為範圍是 **共用** 的，所以整個組織都使用數列格式。 您不能為組織的不同部分設定不同的數列格式。

## <a name="performance-considerations-for-number-sequences"></a>數列的效能注意事項

在設定數列之前，請考慮以下有關數列設定如何影響系統效能的資訊。

### <a name="continuous-and-non-continuous-number-sequences"></a>連續和非連續數列

數列可以是連續的或不連續的。 連續的數列不會跳過任何數字，但數字可能不會按順序使用。 非連續數列中的數字按順序使用，但數列可能會跳過數字。 例如，如果使用者取消交易，則會產生一個數字，但不會使用。 在連續的數列中，該數字稍後會被回收。 在非連續數列中，不使用數字。

外部文件 (例如訂購單、銷售訂單和發票) 通常需要連續數列。 但是，連續數列會對系統回應時間產生不利影響，因為每次建立新文件或記錄時，系統都必須從資料庫中請求一個編號。

如果您使用非連續數列，您可以在 **數列** 頁面的 **效能** FastTab 上啟用 **預指派**。 當您指定要預指派的數字數量時，系統會選取這些數字並將它們儲存在記憶體中。 只有在使用了預先指派的數量後，才會從資料庫中請求新號碼。

除非法規要求您使用連續數列，否則我們建議您使用非連續數列以獲得更好的效能。

### <a name="automatic-cleanup-of-number-sequences"></a>自動清理數列

在電源故障、應用程式錯誤或其他意外故障的情況下，系統無法自動回收編號以實現連續數列。 您可以手動或自動執行清理過程以恢復遺失的號碼。

計劃清理過程時，請仔細考慮伺服器使用情況。 我們建議您在非高峰時段以批次處理作業的形式執行清理。
