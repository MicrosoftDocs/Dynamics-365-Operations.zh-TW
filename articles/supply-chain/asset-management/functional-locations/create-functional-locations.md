# <a name="create-functional-locations"></a>建立功能位置

[!include [banner](../../includes/banner.md)]

 

本主題說明如何在資產管理中建立功能位置。

建立功能位置結構時，請注意功能位置一旦建立，就無法移開原始位置。 因此建議您先審慎思考功能位置的結構，之後再於資產管理中建立。 若您不滿意某個功能位置，且其尚未投入使用，則您可將之刪除。

若要開始使用功能位置，請先建立兩個功能位置的「類別」：

- 建立 *一個* 無子位置的預設功能位置。 此功能位置僅可用於您新建資產的標準位置。  
- 建立公司內維護作業管理所必備的功能位置結構。

## <a name="create-a-default-functional-location"></a>建立預設功能位置

使用功能位置時，請先建立一個新建資產可使用的預設位置。 此功能位置可由 **資產管理** > **設定** > **資產管理參數** > **資產** 連結 > **預設功能位置** 欄位來選擇。 您建立新資產且未針對這些資產設定功能位置結構時，即可使用此預設功能位置。

1. 選擇 **資產管理** > **通用** > **功能位置** > **所有功能位置**。  
2. 在 **所有功能位置** 中，選擇 **新增**。
3. 在 **功能位置** 欄位插入識別碼，例如以「0000」或「預設」代表這是特殊的功能位置。
4. 在 **名稱** 欄位插入預設功能位置的名稱。
5. 請 *勿* 於 **上層** 欄位中選擇上層 – 將此欄位保留空白。
6. 在 **功能位置類型** 欄位中，選擇要作為預設功能位置的功能位置類型。 請參閱[功能位置類型](../setup-for-functional-locations/functional-location-types.md)取得如何設定功能位置類型的詳細資訊。
7. 選擇 **確定**。 建議不要在此功能位置新增更多資料，因為這只是新資產安裝到公司功能位置前暫時使用的位置。

## <a name="create-functional-locations"></a>建立功能位置

下列程序描述如何建立公司維護管理作業所必備的功能位置。

1. 選擇 **資產管理** > **通用** > **功能位置** > **所有功能位置**。 您可用網格檢視或詳細資料檢視來建立功能位置。
2. 選擇 **新增** 按鈕。
3. 在 **功能位置** 欄位插入識別碼。
4. 在 **名稱** 欄位插入功能位置的名稱。
5. 若該功能位置為結構中的子位置，請於 **上層** 欄位中選擇上層位置。
6. 於 **功能位置類型** 欄位選擇類型。
7. 選擇 **確定**。
8. 選擇功能位置，並按一下 **編輯** 按鈕以新增資訊。

>[!NOTE]
>依據功能位置生命週期狀態的設定，您可能需要建立功能位置的所有子位置，然後變更功能位置生命週期狀態，之後再開始安裝資產。 請參閱[在功能位置安裝資產](../functional-locations/install-objects-on-functional-locations.md)取得資產安裝的詳細資訊。 請參閱[功能位置生命週期狀態](../setup-for-functional-locations/functional-location-stages.md)，深入瞭解功能位置生命週期狀態的設定。

詳細資料檢視會有 FastTabs，您可在此新增和編輯功能位置的資訊。

## <a name="general-information"></a>一般資訊

本節概述功能位置結構的上層與下層的資訊。 **詳細資料** 區段會顯示資產屬性數量、維護計畫和該功能位置相關的資產。 在 **庫存** 區段中，您可選擇該功能位置相關的站點和倉庫。 站點和倉庫會搭配工單項目預測使用。 建立項目預測時，會自動使用資產功能位置的站點和倉庫資訊。 在 **生命週期狀態** 區段中，會顯示功能位置生命週期狀態的資訊。

## <a name="installed-assets"></a>已安裝資產

請參閱[在功能位置安裝資產](../functional-locations/install-objects-on-functional-locations.md)取得資產安裝的詳細資訊。 您可使用此 FastTab 上的 **檢視** 按鈕，以顯示更多欄位。 網格內可呈現 **生效日期** 和 **子資產** 欄位。

## <a name="asset-attribute-requirements"></a>資產屬性要求

在此 FastTab 上，您可新增此功能位置上安裝的資產的特定屬性要求。 這些要求僅供參考。 這些要求不會阻止您安裝具備其他屬性要求的資產。 選擇 **新增明細**，並選擇屬性類型。 然後插入相關 **值**、在 **閾值標準** 欄位選擇閾值，然後儲存記錄。

## <a name="maintenance-plans-and-maintenance-rounds"></a>維護計畫和維護回合

您可在此新增功能位置的維護計畫和維護回合，包括開始日期。 功能位置上安裝的資產可能已設有其他維護計畫。 所有維護計畫和維護回合都可用於排程功能位置的資產日曆項目，以及排程目前安裝的資產。

>[!NOTE]
>若您在排程維護計畫後至 **所有功能位置** 詳細資料檢視 > **維護計畫** FastTab 更新資產類型、資產品牌和資產模型的設定，則該功能位置現有的維護排程項目會自動刪除。 若要依據功能位置上更新後的維護計畫建立新的排程項目，您必須為該功能位置執行新的維護計畫排程。 

## <a name="address"></a>地址

在 **地址** FastTab 插入功能位置的地址。 功能位置上的地址會繼承，若子位置未定義地址，則會使用上層位置的地址。

## <a name="workers"></a>工作者

在此 FastTab 上，您可新增隸屬於此功能位置的工作者，並為該工作者選擇主要的功能位置。 

## <a name="attributes"></a>屬性

在此 FastTab 上，您可設定功能位置屬性的值。 這些屬性可用於描述功能位置的屬性或特性，例如結構屬性、建構類型、區域說明或地上或地下位置。

選擇 **新增明細**，並選擇屬性類型。 下一步，插入與屬性類型相關的 **值** 並儲存記錄。

## <a name="financial-dimensions"></a>財務維度

您可為功能位置選擇財務維度。 可設定[功能位置類型](../setup-for-functional-locations/functional-location-types.md)，允許從功能位置自動更新財務維度。 這代表財務維度上安裝的資產會自動取得該功能位置的財務維度。 若您想要在不同位置設立成本中心，此特性非常實用。

上層功能位置的 **站點**、**倉庫**、**地址** 和 **財務維度** 等資料更新時，您可選擇相應更新相關子功能位置。 將開啟對話方塊，為您提供更新選項。

## <a name="copy-a-functional-location-structure"></a>複製功能位置結構

若您公司的數個功能位置皆具備類似的位置結構，則可使用資產管理內的複製功能，快速建立數個相似的位置階層。 複製特定功能位置或整體結構時，新的位置或結構名稱會與您所複製的相同。 複製程序完成後，只要新的功能位置選擇的功能位置生命週期狀態允許，您即可在新的功能位置輕鬆變更名稱或其他設定。

1. 在 **所有功能位置** 中，選擇要複製的功能位置。 例如，若要複製包含子位置在內的整個功能位置結構，請選擇最上層的位置 (上層位置)。
2. 選擇 **複製功能位置結構** 按鈕。 您於清單頁面中選擇的位置會顯示於 **複製來源** 欄位。
3. 在 **新的功能位置** 欄位插入新位置的名稱。
4. 關於 **貼入下方的上層** 欄位中，只有當您建立的位置屬於現有功能位置結構時，才需要插入上層識別碼。
5. 按一下 **確定**。 新的功能位置結構會顯示於 **所有功能位置**。

>[!NOTE]
>複製功能位置結構時，新結構的功能位置生命週期狀態會設為功能位置建立「第一個狀態」。 功能位置目前的生命週期狀態，將決定您能否於 **所有功能位置** 使用 **重新命名** 和 **刪除** 按鈕，對功能位置重新命名或刪除。

## <a name="delete-a-functional-location"></a>刪除功能位置

您可連同相關子位置刪除功能位置，前提是您要刪除的所有功能位置上皆未安裝資產，而且目前功能位置生命週期狀態允許刪除動作。

1. 在 **所有功能位置** 中，選擇要刪除的功能位置。
2. 如有需要，請將功能位置更新為允許刪除功能位置的功能位置生命週期狀態。
3. 選擇 **刪除**。

>[!NOTE]
>若您無法刪除功能位置，則可以此為目的來設定功能位置生命週期狀態，即可處理刪除作業。 例如，您可在 **功能位置生命週期狀態** 表單於非使用中階段設定「已報廢」或「已刪除」階段。
